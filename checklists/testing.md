# Testing And Evaluation Checklist

- Acceptance criteria are testable, not interpretive.
- The happy path is covered.
- Important failure paths are covered.
- Previous bugs have regression checks where reasonable.
- Edge cases have been named, even if all are not automated.
- Manual evaluation criteria are explicit.
- Observed behavior matches the spec, not just the implementation.
- Adjacent behavior has been spot-checked for regressions.
- Logs, messages, or visible errors are understandable.
- A second pass of testing has been considered after fixes or refactors.
